Commit messages

Every time you push your changes up to Github, you always have to incluse a commit message that explains in detail the changes you made.

Things to remember when commit a message

  • Minimum of 3 words & 10 characters
  • Spell all works correct & proper grammer.
  • First letter is capitalize with no period.
  • Must start with a verb: Create, Fix, Solve, est...
  • Pretend each commit starts with the phrase; "This commit will..."

Landmark roles

Header Role

The header role is to define an element as a masthead. It is to add to the primary 'header' for Accessibility needs.

Nav Role

This role is to define an element as the primary navigation. It is to add in the top 'Nav'

Main Role

The Main role is to define an element as being the main content of the page. It is added to the 'Main'

Footer Role

The footer role is to define an element as information about the content.

E.g. copyright, terms, privacy policy. It is usually added to the footer.

Font Properties

Font-family

Changed the font-familys in html, h1, h2, h3 in CSS.

Font-size

Adjusting the size of the font: a single number with a unit (rem [the best choice], em, px).

Font-weight

Sets how think or thin characters in text should be displayed.

Font-style

Specifies fonts style for text.

Line-height

For adjusting the line-height: a single number, a multiplier that multiples the font size.

Font-shorthand

Font-shorthand is shown within CSS.
